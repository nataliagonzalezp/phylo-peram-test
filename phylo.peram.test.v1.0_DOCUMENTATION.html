1. Purpose and Overview

This script implements a phylogenetic extension of the peramorphosis test (Piras et al. 2011) to infer heterochronic events across all ancestor-descendant pairs in a phylogenetic tree.

Heterochrony refers to evolutionary changes in the timing or rate of developmental events. This script classifies each phylogenetic branch as:

ClassificationMeaning
PERAMORPHOSIS The descendant has a more developmentally advanced shape than its ancestor (overdevelopment).
PAEDOMORPHOSIS The descendant retains a more juvenile-like shape than the ancestor (underdevelopment / retention of juvenile features).
NC No Change — the difference between distances is below the specified threshold; the branch is not classified as heterochronic.

The method uses geometric morphometric data (landmark coordinates) and a developmental progress indicator extracted from a TPS file, combined with a phylogenetic tree. For each species, the expected shape at maximum developmental time is estimated via Procrustes regression. Ancestral shapes are reconstructed using Maximum Likelihood on phylogenetic PCA scores (via geomorph's gm.prcomp). Bootstrap resampling is then used to assess the robustness (recovery frequency) of each inferred pattern.

The script accepts both pre-aligned and raw (unaligned) landmark coordinates, and can derive the developmental indicator from log(centroid size) when no AGE field is available in the TPS file. It also handles trees without branch lengths by assigning unit lengths and issuing a warning.

The input mode is not inferred automatically from the TPS file — the user must explicitly set the corresponding flags in the script (see Section 4.3). The rule used to define the developmental progress indicator is:

  1. Raw / unaligned TPS without an AGE field: the script aligns landmarks, calculates centroid size from the original unaligned coordinates, applies a log transformation, and uses log(centroid size) as the developmental progress indicator.
  2. Pre-aligned TPS with an AGE field: the script reads aligned coordinates directly and uses the AGE values as the developmental progress indicator.
  3. Pre-aligned TPS without an AGE field: size information has been removed by alignment; the script stops and asks the user to provide an external developmental progress indicator.

By default the script assumes pre-aligned coordinates with an AGE field present (coords_aligned = TRUE, has_age_label = TRUE).

3. Input Files

The script requires two input files placed in the same working directory as the script, or with full paths specified: a TPS file with the coordinates and a reference tree.

3.1 TPS File (.tps)

A standard Thin-Plate Spline file containing landmark coordinates and specimen identifiers. Depending on the flags set in Section 4.3, the file may or may not contain an AGE= field, and coordinates may or may not be pre-aligned.

With AGE label:

LM=10
0.123 0.456
0.789 0.012
... (one row per landmark)
ID=SpeciesName
AGE=0.75

Without AGE label (raw coordinates):

LM=10
0.123 0.456
0.789 0.012
... (one row per landmark)
ID=SpeciesName

Requirements — all cases:

  • Specimen IDs (ID=) must match species names in the phylogenetic tree, or must be modifiable to species names (see Section 10.1).
  • All specimens of the same species must share the same species label.
  • Each species must have at least 2 specimens.

Additional requirements when has_age_label = TRUE:

  • Every specimen must have a numeric value for the developmental indicator (field specified by tps_devtime_keyword, default: AGE).
  • The number of AGE= lines must equal the number of specimens.
  • Each species must have variation in the developmental indicator (not all specimens at the same AGE value).

Additional requirements when coords_aligned = FALSE:

  • Coordinates must be raw (unaligned) landmark data in the original coordinate space. gpagen() will perform Generalized Procrustes Analysis.

3.2 Tree File (.tre)

A phylogenetic tree in Newick format.

  • All tip labels must exactly match species names in the TPS file (after any ID parsing; see Section 10.1).
  • Node labels are automatically removed by the script (not used).
  • Branch lengths are used for ancestral state reconstruction when present. If absent, all are set to 1 and a warning is issued (see Section 9.3).
  • For plotting, equal branch lengths are applied to a local copy only; the calculation tree is never modified.
  • The tree must be fully resolved (binary) for ancestral reconstruction.

4. User-Defined Parameters

These are the only values a user needs to modify before running the script. They are located at the top of the script under the section USER-DEFINED PARAMETERS.

4.1 Input File Names

TPS_file <- "yourfile.tps"
The filename (or full path) of the TPS input file. Change this to match your actual TPS file name.
tree_file <- "yourtree.tre"
The filename (or full path) of the Newick tree file. Change this to match your actual tree file name.

4.2 TPS Field Identifiers

tps_specID default: "ID"
The keyword used in the TPS file to identify specimens. Matches lines of the form ID=SpeciesName. Change only if your TPS file uses a different keyword (e.g., "SPECIMEN").
tps_devtime_keyword default: "AGE"
The keyword used in the TPS file for the developmental/age indicator. Matches lines of the form AGE=0.75. Change this if your TPS uses a different keyword (e.g., "CS", "DEVTIME"). Only used when has_age_label = TRUE.

4.3 Input Mode Flags

coords_aligned default: TRUE
Set to TRUE if the TPS file contains Procrustes-aligned (superimposed) landmark coordinates.
Set to FALSE if the TPS contains raw (unaligned) coordinates — gpagen() will be called automatically.
has_age_label default: TRUE
Set to TRUE if the TPS file contains an AGE= field for each specimen.
Set to FALSE if no such field is present. In this case coords_aligned must also be FALSE, and log(centroid size) from gpagen() will be used as the developmental indicator.

Valid flag combinations:

coords_aligned has_age_label Behavior
TRUE TRUE ✔ Supported — Standard workflow. Pre-aligned coordinates and AGE labels used directly. No alignment performed.
FALSE FALSE ✔ Supported — Raw coordinates, no AGE field. gpagen() aligns coordinates and extracts centroid sizes. log(centroid size) assigned as Devtime.
FALSE TRUE ✔ Supported — Raw coordinates with AGE labels. gpagen() aligns coordinates. Provided AGE values used as Devtime.
TRUE FALSE ✘ Not supported — Script stops with an error asking to provide AGE= labels or set coords_aligned = FALSE.

4.4 Analytical Parameters

resampling_proportion default: 0.66
Proportion of specimens retained per species in each bootstrap replicate (66% by default). Range: a value between 0 and 1 (exclusive). Species with exactly 2 specimens are always kept complete regardless of this value.
diff_ratio_threshold default: 0.10
Threshold for the distance-ratio used to classify branches.
diff_ratio = |dist_A − dist_B| / mean(dist_A, dist_B)
  • diff_ratio < threshold → NC
  • diff_ratio ≥ threshold and dist_A > dist_B → PAEDOMORPHOSIS
  • diff_ratio ≥ threshold and dist_A < dist_B → PERAMORPHOSIS
Increasing this value makes the test more conservative (fewer heterochronic events detected). Decreasing it makes it more sensitive but may increase false positives.
n_iter default: 100
Target number of valid bootstrap replicates. The script will attempt up to max(1000, n_iter × 20) total draws to reach this target. A replicate is invalid if any species loses developmental variation or if a model fails.
set.seed(5422)
Random seed for reproducibility. Change this value to obtain a different random stream, or remove the line to disable reproducibility.

                   "geomorph","ggplot2","ggpubr","ggthemes","gridExtra",
                   "Morpho","phytools","readr","tidyr","tidyverse","viridis"))
  2. Set the working directory
    setwd("path/to/your/folder")
    Or in RStudio: Session > Set Working Directory > To Source File Location.
  3. Edit user-defined parameters
    Open phylo.peram.test.R in a text editor or RStudio. Update the values described in Section 4. Pay special attention to coords_aligned and has_age_label (Section 4.3).
  4. Run the script
    In RStudio: click Source.
    In the R console: 
    source("phylo.peram.test.R")
  5. Check the console output
    The script prints informational messages about the input mode, then bootstrap progress: 
    Bootstrap attempt: N
 -> Valid replicate saved! Total: M
    At the end it prints the results table and a summary of output files.
  6. Inspect output files
    All files are written to the working directory. See Section 8 for descriptions.

6. Analytical Workflow

The script executes the following stages in order:

  • 1
    Clean Environment

    Removes all objects from the R workspace (rm(list = ls())) to ensure a clean start.

  • 2
    Load Libraries

    Loads all required packages.

  • 3
    User-Defined Parameters

    Sets file names, input mode flags (coords_aligned, has_age_label), analytical parameters, and auto-generates output file names based on the chosen values.

  • 4
    Extract Data from TPS File

    Reads landmark coordinates with geomorph::readland.tps(). If has_age_label = TRUE, also parses AGE= lines. Constructs an in-memory geomorph.data.frame (gdf) with fields: Data (coordinates), Devtime (if AGE present), Species, ID.

  • 5
    Align Coordinates & Assign Developmental Indicator

    Behavior depends on coords_aligned and has_age_label:

    • TRUE / TRUE — no action; data used as-is.
    • FALSE / FALSE — gpagen() aligns coordinates; log(centroid size) assigned as Devtime.
    • FALSE / TRUE — gpagen() aligns coordinates; AGE-derived Devtime kept.
    • TRUE / FALSE — script stops with an informative error.
  • 6
    Load Tree

    Reads the Newick tree with ape::read.tree(). Node labels are removed. If edge.length is NULL, all branch lengths are set to 1 and a warning is issued. The resulting phylo_tree is used unchanged for all calculations.

  • 7
    Load and Prepare Data

    Extracts coordinate array, species labels, and specimen names from the gdf. Specimen names are reformatted as SpeciesName_IND_01, _IND_02, etc.

  • 8
    Standardize Developmental Indicator

    Scales Devtime to [0, 1] within each species independently using min-max normalization: scaled = (x − min) / (max − min). A small value (1e-7) replaces any zero to avoid regression issues at the origin.

  • 9
    Sort Data by Species

    Reorders all data arrays so that specimens are grouped by species, ensuring consistent indexing throughout.

  • 10
    Check Tree and Data Match

    Verifies that all tree tip labels are present in the morphometric data. Species present in data but absent from tree generate a warning and are excluded from phylogenetic reconstruction.

  • 11
    Estimate Shared Origin Shape (shape_0)

    Fits a Procrustes regression Data ~ Devtime * SpeciesData across all specimens. Predicts the shared juvenile archetype (shape_0) at Devtime = 0, representing the common ancestral juvenile morphology.

  • 12
    Estimate Maximum Shapes (predicted_shapesMAX)

    For each species, fits an independent Procrustes regression (coords ~ DevtimeTwo) and predicts the shape at the maximum observed developmental time — the expected adult or maximum-development shape.

  • 13
    Reconstruct Ancestral Shapes (AncestralCoordinatesMAX)

    Projects maximum shapes onto a phylogenetic PCA (gm.prcomp with phy = phylo_tree), performing ancestral state reconstruction via Maximum Likelihood on PC scores. Ancestral node shapes are back-projected to landmark space.

  • 14
    Infer Observed Heterochronic Trends

    For each tree edge: retrieves ancestor and descendant shapes, computes dist_A (ancestor → shape_0) and dist_B (shape_0 → descendant max shape), and classifies using classify_heterochrony(). Results stored in DistanceResults.

  • 15
    Observed Tree Plot

    Produces a PDF phylogeny with branches labelled PERA / PAEDO / NC and color-coded (blue / red / gray). Uses equal branch lengths for visual clarity only.

  • 16
    Bootstrap Resampling

    Repeats the analysis n_iter times with random subsamples (retaining resampling_proportion per species, minimum 2). Replicates failing validity checks are discarded.

  • 17
    Aggregate Bootstrap Results

    Counts replicates matching the observed trend per branch. Recovery% = matches / valid_replicates × 100. Writes results and raw bootstrap trends to CSV.

  • 18
    Bootstrap Tree Plot

    Produces a second PDF labelled with heterochronic category and bootstrap recovery percentage per branch.

7. Internal Functions Reference

The script defines six internal functions:

classify_heterochrony(dist_A, dist_B, diff_ratio_threshold)
Purpose

Classifies a single ancestor-descendant pair as PERAMORPHOSIS, PAEDOMORPHOSIS, or NC based on two Procrustes distances.

Arguments
dist_AProcrustes distance from ancestor shape to shared juvenile archetype (shape_0).
dist_BProcrustes distance from shared juvenile archetype (shape_0) to descendant maximum shape.
diff_ratio_thresholdNumeric threshold (same as the global parameter).
Returns

A list with $trend (character) and $diff_ratio (numeric).

Logic
diff_ratio = |dist_A - dist_B| / mean(dist_A, dist_B)
diff_ratio < threshold        →  "NC"
dist_A > dist_B               →  "PAEDOMORPHOSIS"
dist_A < dist_B               →  "PERAMORPHOSIS"
estimate_max_shapes(coords, devtimes, species, max_devtimes)
Purpose

Estimates the predicted landmark shape at maximum developmental time for each species using species-specific Procrustes regressions.

Arguments
coords3D landmark array [landmarks × dims × specimens].
devtimesNumeric vector of scaled developmental indicator values.
speciesFactor of species labels, one per specimen.
max_devtimesData frame with columns $Species and $MaxDevtime.
Returns

Named 3D array [landmarks × dims × n_species] with one predicted shape per species.

Notes
  • Requires at least 2 specimens per species.
  • Requires variation in devtimes within each species.
  • Uses geomorph::procD.lm() and predict() internally.
reconstruct_ancestral_shapes(predicted_shapes, phylo_tree)
Purpose

Reconstructs ancestral landmark configurations at internal tree nodes using phylogenetic PCA with Maximum Likelihood ancestral state estimation.

Arguments
predicted_shapesNamed 3D array of tip (species) max shapes from estimate_max_shapes().
phylo_treePhylogenetic tree (ape "phylo" object).
Returns

Named 3D array [landmarks × dims × n_nodes] — one reconstructed shape per internal node.

Notes
  • Uses geomorph::gm.prcomp(phy = phylo_tree) for simultaneous PCA and ancestral reconstruction.
  • Ancestral PC scores are back-transformed to landmark coordinates.
infer_edge_trends(shape_0, predicted_shapesMAX, AncestralCoordinatesMAX, phylo_tree, diff_ratio_threshold)
Purpose

Iterates over all edges of the phylogenetic tree and classifies each ancestor-descendant pair.

Arguments
shape_0Matrix [landmarks × dims] — the shared juvenile origin shape (Devtime = 0).
predicted_shapesMAXNamed 3D array of tip maximum shapes.
AncestralCoordinatesMAXNamed 3D array of reconstructed ancestral shapes.
phylo_treePhylogenetic tree (ape "phylo" object).
diff_ratio_thresholdNumeric threshold.
Returns

Data frame with one row per tree edge: Ancestor, Descendant, DistanceA, DistanceB, DiffRatio, HeterochronyTrend.

draw_species_sample(inds, devtimes, resampling_proportion, min_keep = 2)
Purpose

Randomly selects a subset of specimen indices for one species during bootstrap resampling.

Arguments
indsInteger vector of specimen indices for one species.
devtimesDevelopmental time values (passed for context; selection is index-based).
resampling_proportionProportion of specimens to retain (0–1).
min_keepMinimum specimens to keep (default: 2).
Returns

Integer vector of selected positions within inds (not global indices).

Notes
  • If length(inds) == 2, all specimens are always returned (no resampling).
  • Effective keep count: max(min_keep, round(n × resampling_proportion)).
prepare_plot_variables(results, phylo_tree)
Purpose

Adds plotting-specific columns to the results data frame before tree visualization.

Returns

Same data frame with additional columns: DescNum, AncNum, TrendAbbrev ("PERA" / "PAEDO" / "NC"), TrendColor ("blue" / "red" / "gray70").

plot_heterochrony_tree(phylo_tree, results, pdf_name, plot_title, show_recovery = FALSE)
Purpose

Produces a PDF file with the phylogenetic tree annotated by heterochronic classification per branch.

Arguments
phylo_treePhylogenetic tree with true branch lengths.
resultsResults data frame including TrendAbbrev and TrendColor.
pdf_nameOutput PDF filename (character).
plot_titleTitle displayed on the plot (character).
show_recoveryLogical — if TRUE, displays bootstrap recovery % on each branch.
Output

16 × 10 inch PDF with colored edge labels (PERA blue, PAEDO red, NC gray), DiffRatio values, optional Recovery%, node circles, and legend.

Important

Equal branch lengths are applied to a local copy of the tree for plotting only. The original phylo_tree is never modified.

9. Important Rules

9.1 Bootstrap Recovery Rule

Recovery is always calculated relative to the observed trend.

  1. The observed trend (from the full dataset) is determined first.
  2. Each valid bootstrap replicate is classified independently.
  3. A replicate "matches" if it produces the same trend as observed.
  4. Recovery% = (number of matching replicates) / valid_replicates × 100

A branch classified as NC observed will have 100% recovery if all bootstrap replicates also return NC. Recovery reflects consistency of the pattern, not strength of the signal.

9.2 Tree Integrity Rule

The original phylo_tree (read from the Newick file) is used for all calculations: ancestral reconstruction, edge iteration, and bootstrap.

Equal branch lengths are only used inside plot_heterochrony_tree() for visual clarity, applied to a local copy. The original tree is never overwritten.

9.3 Tree Branch Length Rule

If the Newick file does not contain branch lengths (edge.length is NULL), the script automatically sets all branch lengths to 1 and issues:

Warning
"The tree does not contain branch lengths. All branch lengths have been set to 1. The analysis will proceed with an ultrametric tree (equal unit branch lengths)."

The analysis continues, but ancestral reconstruction with equal branch lengths may not reflect true evolutionary distances. Providing a tree with meaningful branch lengths (divergence times or molecular distances) is strongly recommended when available.

9.4 Bootstrap Validity Rules

A bootstrap replicate is discarded if any of the following occur:

  • Any species has fewer than 2 specimens after resampling
  • Any species has no variation in DevtimeTwo after resampling
  • The Procrustes regression (procD.lm) fails
  • Origin shape prediction fails
  • Maximum shape estimation fails for any species
  • Ancestral PCA reconstruction fails or returns no ancestral states
  • The replicate does not produce a result for every branch

Species with exactly 2 specimens are never resampled — all their specimens are always included in every replicate.

The script attempts up to max(1000, n_iter × 20) total draws. If fewer than n_iter valid replicates are obtained, a warning is issued and recovery percentages use the actual count as denominator.

10. Customization

10.1 Modifying Species Name Extraction from TPS IDs

By default the script uses the TPS ID directly as the species name (Species <- ID). To modify, edit the Species line inside extract_tps_input_data():

# Remove a trailing individual suffix:
Species <- sub("_IND_[0-9]+$", "", ID)

# Take the first two underscore-delimited parts:
Species <- sapply(strsplit(ID, "_"), function(x) paste(x[1:2], collapse="_"))

Modified species names must exactly match the tree tip labels.

10.2 Changing the Developmental Indicator Keyword

If your TPS file uses a keyword other than AGE:

tps_devtime_keyword <- "CS"   # or "DEVTIME", "AGE_DAYS", etc.

Only used when has_age_label = TRUE.

10.3 Using log(centroid size) as Developmental Indicator

If your TPS has no AGE field, set:

coords_aligned <- FALSE
has_age_label  <- FALSE

The script runs gpagen(), extracts centroid size from gpagen()$Csize, and computes log(centroid size) as Devtime. This mode is appropriate for growth series where body size is a reliable proxy for developmental stage.

Note: requires raw (unaligned) coordinates. If coordinates are already aligned, size information has been removed and cannot be recovered.

10.4 Adjusting the Classification Threshold

ValueEffect
0.05More sensitive — more branches classified as heterochronic
0.10Default — balanced sensitivity / specificity
0.15More conservative
0.20Very conservative

Run the script with multiple values to assess robustness of results.

10.5 Adjusting Resampling Proportion

ValueEffect
0.50Retain 50% — more variation, more potential failures
0.66Default
0.75Retain 75% — closer to full dataset

For datasets with very small species samples (3–5 specimens), higher values (0.75–0.80) are recommended to avoid frequent replicate failures.

11. Common Errors and Troubleshooting

Error
"coords_aligned = TRUE but has_age_label = FALSE"
Cause
Pre-aligned coordinates but no developmental indicator — unsupported combination.
Solution
Either provide a TPS file with AGE= labels and set has_age_label = TRUE, or set coords_aligned = FALSE to use log(centroid size).
Error
"The number of AGE values does not match the number of specimens"
Cause
Different number of AGE= lines and specimen blocks in the TPS file.
Solution
Check for specimens missing the AGE= field. Ensure tps_devtime_keyword matches exactly the keyword in your TPS file.
Error
"The following tree tips are missing from the morphometric data"
Cause
One or more tip labels in the tree are absent from the TPS species names.
Solution
Check for spelling differences, extra spaces, or case mismatches between tree tip labels and TPS IDs (after any Species extraction logic).
Error
"Species X has fewer than 2 specimens"
Cause
A species has only one specimen.
Solution
Remove that species from the TPS file and tree, or add specimens.
Error
"Species X has no variation in Devtime/AGE"
Cause
All specimens of a species share the same AGE value (or the same centroid size in log(CS) mode).
Solution
Remove that species or provide specimens with varying developmental times.
Error
"No valid bootstrap replicates were obtained"
Cause
All bootstrap attempts fail validity checks — typically species too small to survive subsampling.
Solution
Increase resampling_proportion (e.g., 0.80 or 0.90). Ensure all species have at least 3 specimens if resampling is expected.
Warning
"The tree does not contain branch lengths. All branch lengths have been set to 1. The analysis will proceed with an ultrametric tree."
Cause
The Newick file does not include branch lengths.
Action
Analysis continues with unit branch lengths. Provide a tree with meaningful branch lengths (divergence times, molecular distances) for more reliable ancestral reconstruction.
Warning
"Only N valid bootstrap replicates were obtained out of the requested M"
Cause
Some replicates failed validity checks.
Action
Recovery percentages remain valid (denominator = N). Consider increasing resampling_proportion or n_iter.
Issue
Bootstrap runs very slowly
Cause
Large dataset or high replicate failure rate.
Suggestion
Reduce n_iter for exploratory runs (e.g., n_iter <- 10). Increase resampling_proportion to reduce failures.

12. Glossary

Ancestor-descendant pair
A pair of nodes in the phylogenetic tree connected by a single branch. The ancestor is the internal node; the descendant is either a tip (species) or another internal node.
Bootstrap recovery
The percentage of valid bootstrap replicates that reproduce the same heterochronic classification as the observed (full-dataset) analysis for a given branch.
Centroid size
A measure of overall landmark configuration size, defined as the square root of the sum of squared distances from each landmark to the centroid of the configuration. Extracted by gpagen() for each specimen. Used as a proxy for developmental stage when has_age_label = FALSE.
DevtimeTwo
Internal variable name for the scaled developmental indicator used in regressions. Derived from raw Devtime values after per-species min-max normalization.
diff_ratio
The key classification metric: |dist_A − dist_B| / mean(dist_A, dist_B). Measures the relative difference between the two Procrustes distances used to classify heterochrony.
dist_A
Procrustes distance from the ancestor's maximum shape to the shared juvenile archetype (shape_0). Represents how far the ancestor developed beyond the juvenile stage.
dist_B
Procrustes distance from the shared juvenile archetype (shape_0) to the descendant's maximum shape. Represents how far the descendant developed beyond the juvenile stage.
Geomorph data frame (gdf)
A specialized list structure from the geomorph package holding landmark coordinates together with associated variables (Species, Devtime, ID) for use in procD.lm(). The Devtime field is present only when has_age_label = TRUE or after gpagen() assigns it.
gpagen()
Generalized Procrustes Analysis function from the geomorph package. Aligns raw landmark configurations by removing differences in size, position, and orientation. Returns aligned coordinates and centroid size per specimen (gpagen()$Csize), used as the developmental indicator when has_age_label = FALSE.
Heterochrony
Evolutionary change in the timing or rate of development. Here assessed as differences in how far a lineage's adult shape deviates from a shared juvenile reference.
log(centroid size)
The natural logarithm of centroid size, used as the developmental indicator when coords_aligned = FALSE and has_age_label = FALSE. Log-transformation stabilizes variance and linearizes the relationship between size and developmental stage.
NC (No Change)
Branch classification where the distance ratio falls below the threshold — the difference in developmental offset between ancestor and descendant is too small to be classified as heterochronic.
Paedomorphosis
Heterochronic pattern where the descendant retains more juvenile-like features relative to its ancestor (dist_A > dist_B).
Peramorphosis
Heterochronic pattern where the descendant shows a more developmentally advanced shape than its ancestor (dist_B > dist_A).
Procrustes distance
A measure of shape difference between two landmark configurations after generalized Procrustes alignment (removing size, position, and orientation differences).
shape_0
The shared juvenile archetype: the predicted landmark configuration at DevtimeTwo = 0, estimated from a Procrustes regression with species interaction across all specimens.
TPS file
Thin-Plate Spline file format — a standard plain-text format for storing landmark coordinate data in geometric morphometrics.
Ultrametric tree
A phylogenetic tree where all root-to-tip path lengths are equal. When a tree without branch lengths is provided, the script assigns unit branch lengths to all edges and proceeds as if the tree were ultrametric.
